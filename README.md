# GreenCarry

상점 주문, 장바구니, 결제, 주문 확인, 리뷰, 고객센터, 관리자 운영 기능을 하나의 흐름으로 연결한 웹 서비스 프로젝트입니다.  
사용자 관점의 주문 경험과 관리자 관점의 운영 기능을 함께 구현했으며, 결제는 Toss Payments API, 배포는 Vercel과 Render를 활용했습니다.

## 프로젝트 개요

- 사용자 주문부터 결제, 주문 확인, 리뷰, 문의까지 이어지는 서비스 흐름을 제공합니다.
- 관리자 페이지를 통해 상점 정보와 주문 내역을 관리할 수 있는 운영 기능을 제공합니다.
- 프론트엔드와 백엔드를 분리 배포해 실제 서비스 환경을 고려한 구조로 구성했습니다.

## 주요 기능

- **사용자 주문**: 상점과 메뉴를 선택해 주문을 시작할 수 있습니다.
- **장바구니**: 선택한 상품의 수량과 금액을 확인할 수 있으며, 총 결제 금액이 실시간으로 반영됩니다.
- **결제 시스템**: Toss Payments API를 연동해 결제를 진행하고, 성공·실패 결과를 주문 흐름에 맞게 처리합니다.
- **주문 확인**: 주문 완료 후 주문 번호, 주문 내역, 진행 상태, 도착 관련 정보를 한 화면에서 확인할 수 있습니다.
- **리뷰 관리**: 작성한 리뷰를 조회하고 기간별로 필터링할 수 있으며, 리뷰 삭제와 사장님 답글 확인이 가능합니다.
- **고객센터**: FAQ 검색, 1:1 문의 등록, 문의 내역 조회·수정·삭제, 답변 확인 기능을 제공합니다.
- **마이페이지**: 주문 이후 리뷰와 문의 내역 등 사용자 활동을 관리할 수 있는 개인화 영역을 제공합니다.
- **관리자 상점 관리**: 등록된 상점 정보를 조회하고 운영에 필요한 상점 데이터를 관리할 수 있습니다.
- **관리자 주문 관리**: 주문 목록과 주문 상태를 확인하고 주문 데이터를 효율적으로 관리할 수 있습니다.
- **관리자 고객 응대**: 사용자 문의와 주문 관련 이슈를 확인하고 운영 대응이 가능하도록 구성했습니다.

## 서비스 흐름

### 사용자 서비스 흐름

`상점/메뉴 선택 -> 장바구니 확인 -> 결제 진행 -> 주문 완료/상태 확인 -> 리뷰 및 고객센터 이용`

1. 사용자는 상점과 메뉴를 선택해 주문을 시작합니다.
2. 장바구니에서 상품 수량과 총 결제 금액을 확인합니다.
3. 결제 페이지에서 주문 정보를 검토한 뒤 결제를 진행합니다.
4. 주문 완료 후 주문 번호, 주문 내역, 진행 상태를 확인합니다.
5. 이후 마이페이지에서 리뷰를 조회하거나 고객센터 기능을 이용할 수 있습니다.

### 관리자 서비스 흐름

`상점 관리 -> 주문 내역 확인 -> 주문 상태 관리 -> 고객 문의 대응`

1. 관리자는 관리자 페이지에서 상점 정보를 확인하고 관리합니다.
2. 주문 내역 페이지에서 주문 목록과 주문 상태를 확인합니다.
3. 운영 중 발생한 주문 이슈와 고객 문의를 처리합니다.

## 핵심 구현 포인트

### 결제와 주문 데이터의 일관성

- 장바구니 금액, 주문 정보, 결제 금액이 한 흐름으로 이어지도록 설계
- 결제 성공, 실패, 취소에 따라 주문 상태가 올바르게 반영되도록 처리

### 사용자 경험 중심의 주문 UI

- 주문 진행 상태를 시각적으로 확인할 수 있는 타임라인 UI 구성
- 주문 완료 페이지에서 상세 정보, 지도, 상태 메시지를 함께 제공

### 운영 관점의 관리자 기능

- 사용자 화면과 별도로 관리자 운영 기능을 분리
- 상점 관리와 주문 관리 기능을 구분해 운영 효율과 유지보수성을 고려

### 마이페이지 기능 확장

- 단순 주문 기능에 그치지 않고 리뷰와 고객센터 기능까지 연결
- 사용자 활동 이력을 확인하고 관리할 수 있도록 구성

## 기술 스택

| 구분 | 기술 |
| --- | --- |
| Frontend | React, Vite, JavaScript, CSS Modules |
| UI | MUI Icons, SweetAlert2 |
| Communication | Axios |
| Payment | Toss Payments API |
| Deployment | Vercel, Render |

## 배포 구성

- Frontend: Vercel
- Backend: Render
- External Service: Toss Payments API

프론트엔드와 백엔드를 분리 배포하고, 환경 변수를 통해 API 서버 주소를 관리하는 구조로 운영 환경에 대응했습니다.

## 환경 변수

프론트엔드 코드에서는 아래 환경 변수를 사용합니다.

```env
VITE_BACKSERVER=YOUR_BACKEND_SERVER_URL
```

## 프로젝트 구조

현재 저장소에는 일부 프론트엔드 화면 관련 파일이 포함되어 있으며, 확인 가능한 구조는 아래와 같습니다.

```text
creen_proejct/
├─ README.md
├─ README.readme
└─ creen_proejct_front/
   └─ src/
      └─ pages/
         ├─ mypage/
         │  └─ user/
         │     ├─ UserCS.jsx
         │     └─ UserReviewList.jsx
         └─ order/
            └─ CheckoutPage.module.css
```

## 현재 코드에서 확인 가능한 기능

- `UserCS.jsx`
  - FAQ 조회
  - 1:1 문의 등록
  - 문의 내역 조회, 수정, 삭제, 답변 확인
- `UserReviewList.jsx`
  - 내 리뷰 목록 조회
  - 기간 필터링
  - 리뷰 삭제
  - 사장님 답글 확인
- `CheckoutPage.module.css`
  - 주문 완료 카드
  - 주문 진행 타임라인
  - 지도 영역
  - 주문 상세 정보
  - 친환경 기여도 UI

## 프로젝트 특징

- 결제 API 연동을 포함한 실제 서비스형 주문/결제 흐름을 구현했습니다.
- 사용자 기능과 관리자 기능을 함께 고려해 서비스 운영 관점까지 반영했습니다.
- 프론트엔드와 백엔드를 분리 배포하며 실제 운영 환경에 가까운 구조를 경험했습니다.
